Programming exercises

This assignment is for practicing programming of concurrent and distributed algorithms, by implementing a few algorithms discussed in class using Python and DistAlgo, a Python extension for programming distributed algorithms.

Programming concurrent algorithms

The task for this part is to implement two algorithms for shared-memory mutual exclusion, (1) Lamport's fast mutual exclusion algorithm and (2) Lamport's bakery algorithm, using Python.

These algorithms were discussed in class in Week 2, and pseudocode for them are posted under Lectures on the interactive site.

You need also write a driver program for testing the correctness and performance of your implementation of the algorithms.

Your driver program should take a given number of threads, and a given total number of requests randomly distributed among the threads, and run the two algorithms (1) and (2) in turn; it should print out "Running Lamport's fast mutual exclusion algorithm" or "Running Lamport's bakery algorithm", when starting the respective algorithms, and print out "P requesting CS", "P entering CS", and "P exiting CS" at each request of CS, each entrance of CS, and each exit of CS, respectively, by a thread P.

Additionally, to facilitate testing of your programs using our driver code, you must define thread classes "Fast" and "Bakery" for algorithms (1) and (2), respectively; the classes must provide a method "cs" that takes a method object "task" and calls "task" in the critical section; you must also provide a top-level method "setup" that takes the set of all thread objects and the total number of requests and initializes shared variables used by the algorithm.

Your program must run under Python 3.2. Your main program must be named main.py, and must run with a command like the following, where n is the umber of threads, and m is the total number of requests, and print the desired output. 

python.exe main.py n m

Your handin must include a README.txt file that includes explanations and credits as well as your identification (reminder: follow the requirements on the course page). Please also include your test data and anything else you have to show your work.

Clarity of your program is the most important, because it helps ensure program correctness, especially for such nontrivial algorithms we study. As much as possible, your program should be at a high level of abstraction like pseudo-code is. In particular, it should use set queries and quantifications (comprehensions and generator expressions in Python), not loops, when possible.

Programming distributed algorithms

The task for this part is to implement two algorithms for distributed mutual exclusion, (1) Ricart-Agrawala's token-based algorithm and (2) Suzuki-Kasami's token-based algorithm, using DistAlgo.

These algorithms were discussed in class in Week 3, and pseudocode for them are posted under Lectures on the interactive site.

The requirements are the same as those above for programming the two concurrent algorithms, except for the following:

  1. Mentions of "thread" are replaced with "process".
  2. Your program should print out "Running Ricart-Agrawala's token-based algorithm for distributed mutex" and "Running Suzuki-Kasami's token-based algorithm for distributed mutex", when starting the respective algorithms.
  3. The two process classes should be named "RAtoken" and "SKtoken".
  4. Instead of a top-level method "setup", the two process classes must provide a method "setup" that takes the set of other processes and the total number of requests.

Follow this link to obtain DistAlgo. To get you started, follow this link to obtain Lamport's 3n message algorithm in DistAlgo, and this link to obtain Ricart-Agrawala's 2n message algorithm in DistAlgo.
